دليل Custom Webhook

ترسل المنصة طلب POST بصيغة JSON إلى الرابط الذي تحدده. يجب أن يستجيب خادمك بكود 2xx لاعتبار الإرسال ناجحاً.

أنواع الأحداث (event): test · confirmation · reminder · subscription_reminder · subscription_expired

إعداد n8n SaaS (موصى به للإنتاج)

  1. على VPS: انسخ examples/n8n/docker-compose.yml وشغّل n8n (راجع DEPLOYMENT.md)
  2. في n8n عيّن: SUPABASE_URL و SUPABASE_SERVICE_ROLE_KEY
  3. استورد: examples/n8n/mken-whatsapp-saas.workflow.json
  4. فعّل الـ Workflow (Active = ON)
  5. في حقل API URL أعلاه ضع الرابط مع معرّف نشاطك: https://YOUR-N8N-DOMAIN/webhook/mken-whatsapp?tenant=YOUR-SLUG
  6. في Token أدخل مفتاح حماية الـ Webhook (Bearer)
  7. في بوابة الإرسال الفعلية أدناه اختر UltraMsg أو Twilio وأدخل بيانات حسابك
  8. للاختبار أولاً: استورد examples/n8n/mken-whatsapp-test.workflow.json

ملاحظة: سير العمل القديم mken-whatsapp-ultramsg.workflow.json للاستخدام الفردي فقط (متغيرات بيئة ثابتة). للـ SaaS استخدم mken-whatsapp-saas.workflow.json.

للاختبار المحلي بدون n8n: شغّل node examples/custom-webhook-receiver.mjs ثم ضع: http://localhost:3030/webhook

عند التفعيل، سيتم إرسال المتغيرات كمعاملات منفصلة ({{1}}، {{2}}، إلخ) بدلاً من النص الكامل المتصل لتلائم قوالب Meta المعتمدة.

⚙️ تخصيص قوالب رسائل الواتساب
المتغيرات المدعومة:
{brandName}: اسم المنشأة | {customerName}: اسم العميل | {phone}: جوال العميل
{serviceTitle}: الخدمة | {activityTitle}: النشاط/المتجر | {date}: التاريخ | {time}: الوقت
{appointmentId} / {orderId}: معرف الحجز/الطلب | {orderItems}: منتجات الطلب | {hoursBefore}: الساعات قبل الموعد
{reminderLeadText}: مدة التذكير (مثال: غداً) | {googleReviewUrl}: رابط التقييم على جوجل (بعد ربط GBP)
